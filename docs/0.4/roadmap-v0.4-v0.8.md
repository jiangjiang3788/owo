# OWO Memory Brain v0.4 ～ v0.8 计划表

总路线：

```text
v0.4 历史大整理
  ↓
v0.5 可信记忆
  ↓
v0.6 正式注入
  ↓
v0.7 长期陪伴人格
  ↓
v0.8 自我维护 / 梦境消化
```

到 v0.9 完成之前，Memory Brain 仍是可读取、可整理、可预览的新脑；正式 prompt 注入继续由当前选择的旧档案记忆等旧 owner 执行。

## v0.4：历史大整理

目的：让新记忆脑吃完整个过去。

| 版本 | 名称 | 目标 | 约步骤 |
|---|---|---:|---:|
| v0.4.0 | 历史源扫描器 | 扫描全部聊天、消息数、时间范围、旧记忆数量 | 6 |
| v0.4.1 | 历史切片 / 游标 | 把几万条消息切成可续跑 archiveChunks | 7 |
| v0.4.2 | 回填队列 / 断点续跑 | 建 backfillJobs / backfillRuns，支持暂停、继续、重试 | 8 |
| v0.4.3 | 历史事件回填 | chunk → 多条历史事件，保留来源范围 | 8 |
| v0.4.4 | 历史事实回填 | 历史事件 → atomic facts | 8 |
| v0.4.5 | 去重 / 冲突 / 过时事实 | duplicate / obsolete / disputed / merged 生命周期 | 8 |
| v0.4.6 | 全量家族 / graph 重建 | 基于全历史 facts 重建 family 和 graph | 8 |
| v0.4.7 | 全历史长期模型重建 | 重建用户画像、AI 自我、世界观、项目脑、互动偏好 | 7 |
| v0.4.8 | 新旧注入对照 / 接管演练 | 对同一输入比较旧注入和新 Memory Brain 预览 | 7 |
| v0.4.9 | 单一 owner 切换门 | 建 formalOwner gate，但默认仍不打开正式接管 | 8 |

阶段完成标准：历史能被分批整理成事件、事实、家族、graph 和长期模型，并能和旧记忆注入做对照。

## v0.5：可信记忆

目的：让记忆可纠错、可解释、可信任。

| 版本 | 名称 | 目标 | 约步骤 |
|---|---|---:|---:|
| v0.5.0 | 记忆审查收件箱 | 汇总低置信、冲突、疑似误记内容 | 6 |
| v0.5.1 | 事实纠错 / 改写 | 用户可改写 fact，保留证据和修改历史 | 7 |
| v0.5.2 | 冲突事实处理 | disputed facts 解释冲突原因并给处理建议 | 7 |
| v0.5.3 | 家族合并 / 拆分 | 用户可合并、拆分、改名 family | 8 |
| v0.5.4 | 长期模型人工修正 | 修正用户画像 / AI 自我 / 世界观 / 项目脑 | 7 |
| v0.5.5 | 纠错影响传播 | fact 修正后传播到 family / graph / model | 8 |
| v0.5.6 | 记忆信任分 | 给事实、家族、模型生成可解释 trust score | 6 |
| v0.5.7 | 可信记忆 gate | fixture 覆盖纠错、回滚、影响传播 | 6 |

阶段完成标准：Memory Brain 不是“记了就算”，而是能被纠正、能解释、能回滚。

## v0.6：正式注入

目的：让新记忆脑开始进入聊天，但必须只有一个正式 owner。

| 版本 | 名称 | 目标 | 约步骤 |
|---|---|---:|---:|
| v0.6.0 | 正式注入 adapter | Memory Brain 通过唯一 adapter 生成 memory block | 6 |
| v0.6.1 | 召回策略调参 | 关键词 + 向量 + graph + 时间线 + 权重召回 | 8 |
| v0.6.2 | 实时注入 trace | 控制台显示为什么注入 / 为什么没注入 | 6 |
| v0.6.3 | 旧系统只读降级 | 旧档案 / 日记 / 向量变成历史来源，不再正式注入 | 7 |
| v0.6.4 | 一键关闭 / 回退 | memoryBrain / legacy / off 三态可回退 | 6 |
| v0.6.5 | 注入质量测试集 | 覆盖漏召回、错召回、重复召回 | 7 |
| v0.6.6 | 正式默认开启候选 | 满足 gate 后允许用户手动开启 | 5 |

阶段完成标准：正式聊天能使用 Memory Brain 注入，而且不会双注入、不会无法回退。

## v0.7：长期陪伴人格

目的：让 AI 不只记事实，还记关系、相处方式和自我连续性。

| 版本 | 名称 | 目标 | 约步骤 |
|---|---|---:|---:|
| v0.7.0 | 关系连续性模型 | 生成双方长期关系节点和关系线 | 7 |
| v0.7.1 | 互动偏好模型 | 记住用户喜欢 / 不喜欢的回应方式 | 7 |
| v0.7.2 | AI 自我日记 | AI 自己的成长记录、反思和版本历史 | 8 |
| v0.7.3 | 情绪节点和纪念日 | 高情绪事件、承诺、和解、纪念日浮现 | 7 |
| v0.7.4 | 主动关心队列 | 未完成线索、需要关心的话题进入候选队列 | 7 |
| v0.7.5 | 人格核版本历史 | 人格核可更新、可解释、可回滚 | 8 |
| v0.7.6 | 陪伴边界设置 | 用户控制主动性、亲密度、提醒频率 | 6 |
| v0.7.7 | 关系回忆 UI | 关系时间线、回忆卡、重要节点房间 | 7 |

阶段完成标准：AI 能保持长期陪伴连续性，而不是只像会查资料的系统。

## v0.8：自我维护 / 梦境消化

目的：让记忆脑会周期性整理、沉淀、浮现和自我修复。

| 版本 | 名称 | 目标 | 约步骤 |
|---|---|---:|---:|
| v0.8.0 | 夜间整理计划 | 低成本定时维护计划和手动夜间整理入口 | 6 |
| v0.8.1 | dream digest | 把一段时期的事件和情绪压成梦境消化摘要 | 8 |
| v0.8.2 | 长期模型周期重建 | 定期重建用户画像、AI 自我、世界观、项目脑 | 7 |
| v0.8.3 | 衰减可解释化 | 解释为什么某条记忆沉底或浮现 | 6 |
| v0.8.4 | 未完成线索复盘 | 定期检查 openThreads / 未完成线索 | 7 |
| v0.8.5 | 周年和关系回忆 | 重要日期、承诺、项目节点自动浮现 | 6 |
| v0.8.6 | 自我修复 gate | 检查错合并、重复、漂移和模型长歪 | 8 |
| v0.8.7 | 低成本后台维护 | 把维护任务做成可暂停、可预算、可恢复队列 | 8 |

阶段完成标准：Memory Brain 像一个会睡觉、会回想、会整理自己的长期小脑。
