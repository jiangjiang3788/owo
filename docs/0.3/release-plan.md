# v0.3 Memory Brain Release Plan

## 总目标

建立适合个人小手机 AI 陪伴 App 的长期记忆脑，不用固定分类，而是让 AI 从聊天中自动生成事件、事实、家族、graph、人物画像、AI 自我和世界观。

## 版本路线

| 版本 | 目标 | 是否替换旧系统 |
|---|---|---|
| `v0.3.0` | 架构骨架、App 入口、旧来源扫描、替换路线 | 否，影子模式 |
| `v0.3.1` | 事件摘要层和真实时间线 | 否，旧系统仍正式注入 |
| `v0.3.2` | 原子事实层、复合记忆拆事实 | 否 |
| `v0.3.3` | 记忆家族、相似度聚类、家族摘要 | 否，但可以开始对照 |
| `v0.3.4` | graph 关系网和多家族归属 | 否 |
| `v0.3.5` | 人物画像、AI 自我、世界观、项目脑 | 否 |
| `v0.3.6` | 注入预览和影子注入对照 | 否，开始判断可替换性 |
| `v0.3.7+` | 新记忆脑成为唯一注入 owner，旧系统降为只读来源 | 是，需 gate 通过 |

## 避免双系统策略

- 新记忆脑在 `v0.3.0`～`v0.3.5` 只扫描、整理和预览。
- 旧回忆日记、记忆表格、向量记忆继续维持当前用户可用能力。
- `v0.3.6` 开始做 shadow injection：新系统生成注入包，但不影响正式请求。
- 只有当控制台连续证明新系统能稳定召回、不过度注入、不漏核心记忆，才切换正式 owner。

## 历史整理策略

1. 先扫描聊天原文、回忆日记、记忆表格和向量记忆。
2. 按聊天对象、时间范围、消息数量分批生成事件。
3. 从事件拆原子事实。
4. 对事实做 embedding 和关键词索引。
5. 相似事实进入记忆家族，一条事实可进多个家族。
6. 家族达到阈值后生成或更新摘要。
7. 建立 graph link：人物、主题、目的、情绪、项目。
8. 每个批次写入控制台，支持回滚。
